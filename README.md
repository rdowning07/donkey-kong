# Donkey Kong

A from-scratch implementation of Donkey Kong in Python and Pygame.
Built as Lesson 3 of a structured game development curriculum — written by hand, not generated by AI.

---

## What This Is

This is not a tutorial clone. Every line of this code was written by the developer as part of a deliberate learning curriculum progressing from Pong → Asteroids → Donkey Kong → Mario Bros → Castlevania → Symphony of the Night.

The goal is first-principles understanding: knowing not just what the code does, but why each piece exists.

---

## What the Code Does

### `main.py`

The game entry point. Initializes pygame, creates the window, runs the game loop. Contains collision detection functions, game state management, and score tracking.

### `player.py`

The player. A rectangle drawn with `pygame.draw.rect()`. Will eventually jump.

### `constants.py`

Shared constants for screen dimensions, gravity, and score values.

**The game loop**

```
while running:
    handle input      ← quit, keypresses
    update state      ← gravity, jumps
    screen.fill()     ← wipe canvas black
    draw everything   ← player, platform, enemies
    display.flip()    ← swap back buffer to screen
    dt = clock.tick() ← cap at 60fps, calculate delta time
```
